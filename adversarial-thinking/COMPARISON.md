# SKILL.md vs adversarial-analytical-pipeline.md 对比分析

## 文件定位

| 文件 | 定位 |
|------|------|
| `SKILL.md` | Claude Code 技能包，plan mode 下自动触发 |
| `adversarial-analytical-pipeline.md` | 独立 prompt 模板，手动粘贴使用 |

---

| 维度 | SKILL.md | adversarial-analytical-pipeline.md |
|------|----------|-------------------------------------|
| **格式** | YAML frontmatter + Markdown | 纯 Markdown |
| **主语言** | 英文为主，角色名中文 | 中文为主 |
| **内容范围** | 两个框架：对抗分析 + CS Polymath Architect | 只有对抗分析管道 |

| **输出规范** | ❌ 无固定模板 | ✅ 严格输出模板 |
| **证据强度分级** | ❌ 无 | ✅ 🟢🟡🔴 三级 |

## 核心差异解读

### 1. 认知重置屏障（pipeline.md 独有）

pipeline.md 的关键创新：阶段二分析师**看不到** Bull/Bear/Engineer 的原始论述，只能看 Moderator 的矛盾清单。

**作用**：防止阶段二被阶段一的立场"污染"，保证第一性原理分析的独立性。

SKILL.md 没有这个机制，模型容易在分析时无意识地"偏向"某个角色。

### 2. CS Polymath Architect（SKILL.md 独有）

SKILL.md 包含第二个框架，专门针对技术/CS 问题，覆盖：
- 硬件层（分支预测、缓存一致性）
- 内核层（VFS、调度器）
- 代码层（编译器优化、无锁数据结构）
- AI/数学层（自动微分、流形学习）

pipeline.md 没有这个框架，技术决策分析能力相对弱。

### 3. 矛盾分类精度

pipeline.md 将矛盾分为四类，帮助决策者知道**如何解决**矛盾：
- 事实性矛盾 → 找数据
- 假设性矛盾 → 澄清前提
- 价值性矛盾 → 决策者自行判断
- 伪矛盾 → 统一定义

SKILL.md 只说"找矛盾"，没有指导如何处理。

---

## 综合结论

**SKILL.md 赢在**：自动触发、覆盖技术场景、上手门槛低。

**pipeline.md 赢在**：输出质量更稳定、认知隔离更严格、矛盾分类更精细。

**建议**：两个文件各有用途，不是非此即彼。理想方案是将 pipeline.md 的三项核心机制（认知重置屏障 + 矛盾分类 + 输出模板）合并进 SKILL.md，同时保留 CS Polymath 框架。
